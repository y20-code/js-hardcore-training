PRD 需求文档：武器铁匠铺锻造系统
需求背景：
在“Word Killer”游戏的城镇里，有一个铁匠铺。玩家可以拿着初始武器和一堆材料去找铁匠升级武器。我们需要你在前端构建这个铁匠铺的底层系统。

你的研发任务：
请在代码里创建一个纯对象，名字必须叫 forgeSystem。这个对象需要包含以下内容：

1. 店铺基础信息（静态属性）
铁匠名字叫 smithName，值为 "欧冶子"。

店铺等级叫 shopLevel，初始值为 1。

2. 核心业务动作：升级武器 (upgradeWeapon)
玩家点击“锻造”按钮时，会触发这个动作。

输入要求（参数接收）：

第一个参数是武器的基础攻击力（必传参数，数字）。

第二个参数是锻造手续费。如果系统没传或者传了 undefined，必须默认为 50 金币。

后面跟着的是一堆强化材料的能量值（比如 10, 20, 15...）。玩家可能放 3 块材料，也可能一块都不放。你需要把它们全部打包收起来。

内部计算要求：

你需要算出所有材料的总能量值。（提示：如果没放材料，总能量必须是 0，不能报错）。

算出最终攻击力。公式：最终攻击力 = 基础攻击力 + 材料总能量值 + (店铺等级 * 10)。（注意：这里必须要读取铁匠铺自己的真实店铺等级）。

输出交付要求：

锻造完成后，必须**返回一个装备卡片（对象）**给背包系统。

卡片里包含两个字段：finalAttack (算出来的最终攻击力) 和 cost (本次消耗的手续费)。

3. 核心业务动作：全服广播打广告 (broadcastAd)
系统会定时调用这个动作，让铁匠铺在游戏世界频道喊话。

业务要求：

不需要接参数，不需要返回结果。

动作触发后，必须等待 2 秒钟。

2 秒钟后，在控制台打印出一句话：“我是 欧冶子，我的铁匠铺目前是 1 级，欢迎来打造神器！”（必须动态读取铁匠自己的名字和等级）。

PM 的业务需求文档（PRD）：违规词汇监控与封号功能
请你创建一个对象 const chatMonitor: ChatMonitor = { ... }，实现以下业务：

1. 基础配置

系统名 systemName 初始化为 "天网监控"。

重点监控词 bannedWord 初始化为 "挂机"。

2. 核心业务动作：扫描玩家聊天记录 (scanMessages)
当玩家在公屏发了一堆话时，后台会调用这个动作。

输入参数要求：

第一个参数是替换字符。如果没传，默认用 "*"。

后面跟着不确定数量的聊天句子（比如 "今天真开心", "有人挂机吗", "打得不错"）。全部打包收好。

内部计算要求（重点，完全不同于之前）：

找坏人：你需要遍历这些聊天句子，找出有多少句话里包含了 this.bannedWord（也就是"挂机"）。统计出违规句子的总数量。（提示：可以用 filter 配合 includes，或者 forEach 统计）。

净化环境：把那些没有违规的句子挑出来，并且在每句话前面加上系统签章。比如把 "今天真开心" 变成 "[天网监控]审核通过: 今天真开心"。（提示：需要读取 this.systemName）。

输出交付要求：

返回一个对象，包含 threatCount (违规句子总数) 和 safeMessages (净化后加上签章的安全句子数组)。

3. 核心业务动作：启动巡逻 (startPatrol)

不需要接参数，不需要返回结果。

触发后，等待 1 秒钟。

1 秒后打印：“[天网监控] 启动完毕，正在严厉打击包含 [挂机] 的发言！”（必须动态读取系统名和违规词）。
